Сегодня работаю над пассивным сбором информации OSIN.
Пассивный сбор информации - это как правило, сбор информации без прямого нелегального взаимодействия с целью, либо легального на пример как пользователь.
Цель пассивного сбора информации как правило получить такую информацию которая увеличить количество векторов атаки или расширить существующие.

Утилиты:
	whois - получает регистрационные сведения о домене (регистрате), получает сведения о регистрате (домена которого зарегистрировали) и регистраторе (домена который зарегистрировал). На пример регистратор в google.com это https://www.markmonitor.com/ а у https://www.markmonitor.com/ нет регистратора, получается это вроде как конечная точка в цепи регистрат->регистратор. 
Что дает нам whois:
	1. Данные владельца домена (registrant).
	2. Данные компании которая зарегистрировала домен (registrator)
	3. Дату когда был создан домен и до какого времени оплачен.
	4. Данные сервером отвечающие за работы DNS, для этого имени.
	5. Текущее состояние домена, активен или не активен.
Интересная информация пишут что частые обращения к базе whois могут ввызывать бан, то есть для частых опросов нужна какая то задержка в цикле как я понимаю.

Большой фреймворк для работы по сбору информации, много модулей нужно узнать какие самые популярные для использования. Поробую получить API key для популярных модулей.  

 dnsrecon
Хороший инструмент для разведки. Скачал словари для режима brt https://github.com/danielmiessler/SecLists.
Вообщем сделаю заметки по режимам:
	std - стандартная разведка поиск слабо защищенных узлов для дальнейшей атаки.
	btr - поиск поиск 
	r - поиск забытых ресурсов по диапазону адресов
	srv - поиск точек входа на конкретные порты, можно узнать в какой порт ломиться и что на этом порту работает kerberos на пример.
	snoop - нужен в основном для фишинга, поиск информации в cache публичного DNS сервера.
